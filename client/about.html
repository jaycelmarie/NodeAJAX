Application

Nana's Products allow users to Search, Add, Update and Delete products in the database. I used mongodb to store the JSON object of products and user information in the database.

The server is running on localhost/8080 where the html file is sending a request to the server using HTTP modules

The express server then gets the data from the database (mongodb) e.g products, users info.

The server turns the data into a JSON response to send back to the client (HTML) and browser renders it in.



Functionalities

Search

Users can searche for a product by Name, Description or SKU number.

If a match/s found, first 10 products are displayed. The user can then navigate through the products with arrows (pagination)

If no product is found -> 'No product is found!' message.

Alternatively, users can search for products according to price range.

If a product is already searched, it filters it so include within searched products along the price range products.

Add Product

When adding a new product, the user is presented with a form to fill in the details.

Validation: Checks if all fields are filled in, along with checking if SKU is an intenger/number format.

Success: New Product Added Displayed. ----- Failed: Error Message Displayed.

Update Product

When updating a product, the user searches up the product name, if a product is found. The product details is show to user and they are able to make changes. Once submitted, it checks if the SKU if integer/number before successful changes.

Success: Product is updated Display Message. ----- Failed: Error Message Displayed.

Delete Product

When deleting a product, a search bar is prompted to user to enter the product's SKU number.

Success: Product is deleted Display Message. ----- Failed: Error Message Displayed.

*When searching for the deleted product, it will show as 'No Products Found!'

User Login/Logout

Users can login/logout. A pop up container renders in to prompt for user logins. Once authenticated and succesful, it logs in the user with a 'Welcome, -username-'.

Users can log in using: username/email and password

The server then checks if the user is logged in, if yes, it displays the 'Logout' button. the middleware checks if user is logged in.

If user decides to log out, the server destroys the session.

Registration

If user does not exist, they can register and the details get saved into the database.

Even if session is destroyed, the successful registered users can maintain their account.

Technologies

Frontend Technologies

HTML (HyperText Markup Language) CSS (Cascading Style Sheets) Javascript (JS) FETCH API
Structuring the content of web pages. Style HTML elements and defining layout of the web page. Used for adding interactivity to web pages, handle user actions and making asynchronous requests to server. Used to make HTTP Reqeusts to fetch or send data asynchronously.

Backend Technologies

Node.js Express MongoDB Express-session Middleware
Javascript runtime envrionment to run the Javascripts on the server-side. Web application framework for node.js to simplify process of building web apps and APIs. Used to store and manage data e.g product, users. Express-session Middleware : Manage user sessions and store session data on the server.

Weaknesses

The application cannot filter results of specific product within the price range. Instead, it searches up all products within that price range, along with the specific product searched by user.

The user has to know the SKU in order to delete a product.
